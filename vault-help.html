Overview

The Character Vault is a roster management tool for Savage Worlds characters. It's designed for game masters, writers, and publishers who need to maintain large collections of NPCs, pre-generated characters, and named personalities across multiple products and regions.

Where the Character Forge builds one character at a time from scratch, the Vault manages the fleet ‚Äî dozens or hundreds of characters across a product line, each with mechanical builds, narrative content, and production status tracking.

The Vault saves automatically to a GitHub Gist (cloud storage) with every edit. Your data persists across sessions, devices, and browser refreshes. No manual saving is required.

The Vault and the Forge share a common JSON format (diceforge-character-v1) for lossless bidirectional transfer. Build in either tool, move freely between them.
Quick Start ‚Üí

Quick Start

On first visit, the Vault prompts for a GitHub Personal Access Token with Gist permissions. This connects the tool to your private cloud storage. You only need to do this once per browser.

Once connected, create characters with + New Character, or import existing builds via ‚áÑ Import. Each character appears in the sidebar roster and can be selected for editing.

The three-column layout shows your roster on the left, the active editing panel in the centre, and the character's stat summary on the right. Click any section heading on the right to jump into its editor.

The Five-Minute Workflow

Create a character. Set attributes and skills via the die stepper buttons. Pick hindrances and edges from the catalogues. Add weapons and armour. Fill in the narrative fields. Check the Build Audit. Export or commit to canon. Done.

Everything saves automatically. There is no save button. The green dot in the header means your data is safely in the cloud.

‚Üê Quick Start Cloud Storage ‚Üí

Cloud Storage (Gist)

The Vault stores all character data in a GitHub Gist ‚Äî a private, version-controlled JSON file hosted on GitHub's infrastructure. This gives you cloud persistence, device portability, and a complete edit history accessible through GitHub's web interface.

Setup

On first use, the Vault asks for a GitHub Personal Access Token. Generate one at github.com/settings/tokens with the gist scope. The token is stored in your browser's localStorage and never sent anywhere except GitHub's API.

How Saving Works

Every edit triggers an automatic save cycle: a 500ms debounce collects rapid changes, then the full roster is serialised and pushed to the Gist. The header sync indicator shows the result. Saves happen silently in the background with no overlay or interruption.

Version History

Because the Gist is a Git repository, every save creates a new revision. You can view the full edit history at https://gist.github.com/<gist-id>/revisions. GitHub stores revisions indefinitely, providing a free backup system.

Need to roll back? View the Gist revision history on GitHub and copy an earlier version of the JSON data.

Creating Characters

Click + New Character in the sidebar to open the creation modal. Fill in the required fields:

NameCharacter's name. Must be unique within the roster.
AncestryDropdown of available ancestries (Human, Dwarf, Elf, etc.).
RegionWhere this character is from ‚Äî Ammaria, Saltlands, Vinlands, Concordium, or Glasrya.

Editing Characters

Select a character from the roster, then click any section heading on the right column to open its editor. Alternatively, click the Edit button in the action bar for the core stats panel, or use the quick-link buttons in the centre column.

All edits save automatically. There is no save button ‚Äî the moment you change a value, it's written to memory and queued for cloud sync. You can edit freely without worrying about losing work.

The ‚úé icon next to section headings indicates that section is clickable and will open an editor panel.

Search & Filtering

The search bar at the top of the roster filters characters by name. Type to filter ‚Äî matches are instant. Clear the search to show all characters.

Filter dropdowns below the search bar let you narrow by Region, Category, and Tier. Filters are cumulative ‚Äî selecting both "Ammaria" and "Wild Card" shows only Wild Card characters from Ammaria.

The Status Dashboard button switches to a table view showing all characters with their completion status in a sortable grid.

Panel ‚Äî Skills

Lists all assigned skills with die stepper buttons. Click + to add a new skill from the dropdown, or type a custom skill name. Skills are sorted alphabetically.

Each skill shows its current die value. Use the ‚àí and + buttons to step through the die chain: d4 ‚Üí d6 ‚Üí d8 ‚Üí d10 ‚Üí d12. Pre-gen characters are capped at d12.

Click the ‚úï button next to a skill to remove it entirely.

Core skills (Athletics, Common Knowledge, Notice, Persuasion, Stealth) typically start at d4 for player characters. For NPCs, only assign the skills that character would actually use ‚Äî there's no point budget for NPCs.
‚Üê Edit Panel Hindrances ‚Üí

Panel ‚Äî Hindrances

Select hindrances from the catalogue dropdown or type a custom hindrance name. Each hindrance is added to the character's list and displayed in the stat summary.

Include the severity in the name for clarity ‚Äî e.g. "Loyal (Minor)" or "Vengeful (Major)". The Build Audit checks hindrance count against Savage Worlds limits.

‚Üê Skills Edges ‚Üí

Panel ‚Äî Edges

Select edges from the catalogue or type a custom edge name. The catalogue contains all core Savage Worlds edges plus any setting-specific edges.

Edges not found in the catalogue are tagged as (legacy) in the stat summary. This flags them for review but doesn't prevent their use ‚Äî it's common when importing characters built under older rulesets or custom content.

‚Üê Hindrances Equipment ‚Üí

Weapons, Armour & Gear

Weapons

Add weapons from the equipment catalogue. Each weapon entry includes name, damage, AP value, and range (if applicable). The catalogue covers all Savage Worlds core weapons.

Weapons appear in the stat summary with full stats and are included in all exports.

Armour

Add armour from the catalogue. Each piece shows name, protection value, and coverage area. When armour is added or removed, the character's Toughness armour bonus is recalculated automatically and the "X (Y)" Toughness display updates in real time.

Gear

General equipment ‚Äî rope, lanterns, trade goods, etc. Enter items as a comma-separated text list. This is a legacy field; characters imported from older data may have weapons and armour listed here as text. If so, add them properly via the structured panels above and clear the gear text.

Items with apostrophes (Sailor's Coat, Knight's Shield) are handled correctly throughout the tool.

Stat Block

The Stat Block panel renders a formatted, read-only view of the character in the standard Savage Worlds stat block format. This is the "what it looks like in the book" view ‚Äî attributes on one line, skills on the next, derived stats, then edges, hindrances, gear, and powers.

Canon Commits

The Canon Commit system provides a formal "sign-off" mechanism for finished characters. When you click üì¶ Commit to Canon, the Vault takes a snapshot of the character's current state and records it as the canonical version.

The commit modal shows a diff of the character's data ‚Äî attributes, skills, edges, hindrances, weapons, armour, derived stats, and equipment funds. Review the diff, then confirm to commit.

Committed characters show a üì¶ badge in the Status Dashboard. The commit timestamp is recorded for audit trail purposes.

Canon Commits are a development workflow tool. The commit data is stored in the Gist alongside the character roster. A future Canon API endpoint will accept these for external validation.
‚Üê Status Dashboard Export Formats ‚Üí

Export Formats

Open the ‚áÑ Transfer panel on any character to access export. Four formats are available, each with Copy to Clipboard and Download File buttons:

JSON (Recommended)

Lossless. Complete character data in diceforge-character-v1 format. The only format that round-trips perfectly between Vault and Forge.

Plain Text

Formatted stat block in standard Savage Worlds layout. For pasting into documents, emails, or chat. Not reliably re-importable.

HTML

Self-contained styled web page. Georgia serif, dark red headings, proper stat block layout. Opens in any browser.

RTF

Rich Text Format. Download gives a .rtf file for Word. Copy pastes as formatted text into Word and other rich-text editors.

All exports include the Pinnacle Fan Licence attribution automatically.
‚Üê Canon Commits Import ‚Üí

Import

Click ‚áÑ Import in the sidebar to open the import modal. Two input modes:

Paste Text

Paste character data directly into the text area. The Vault auto-detects the format: JSON starting with { gets lossless parsing; HTML or RTF markup is stripped and fed through the stat block parser; plain text goes through the stat block parser directly.

Upload File

Click to browse or drag a file. Accepts .json, .txt, .html, and .rtf. The file content is loaded and auto-detected as above.

Preview & Validate

Click üëÅ Preview to parse without committing. The preview shows name, ancestry, tier, category, attributes, skills, edges, hindrances, weapons, and armour. Non-JSON formats display a yellow warning reminding you to review all fields.

Non-JSON imports are best-effort. The stat block parser looks for lines starting with "Attributes:", "Skills:", "Edges:", etc. Weapons and armour listed as text will not be parsed into structured data ‚Äî add them via the Weapons and Armour panels after import.

‚Üê JSON Transfer Troubleshooting ‚Üí

Troubleshooting

Cloud sync indicator is red

A save to GitHub Gist failed. Common causes: expired Personal Access Token, network issues, or GitHub API rate limiting. Check the browser console (F12 ‚Üí Console) for the specific error. If your token expired, generate a new one and re-enter it.

Characters don't appear after refresh

The Vault loads from the Gist on page load. If the Gist is empty or the token is wrong, the roster will be empty. Verify your token has gist scope and the Gist ID is correct. View your Gist directly at https://gist.github.com/<your-gist-id>.

Weapons or armour disappear on reload

Fixed in the current version. Hard-refresh (Ctrl+Shift+R) to load the latest code. Weapons and armour now persist correctly to the Gist.

Build Audit shows red but the character looks right

The audit is calibrated for Novice-rank characters. Higher-rank builds naturally exceed Novice budgets. NPCs and creatures don't follow player creation rules. Read the specific audit message for expected vs. actual values.

Import shows "No attributes or skills found"

The stat block parser expects lines formatted as "Attributes: Agility d6, Smarts d8..." ‚Äî if your source uses a different format, the parser may not recognise it. Use JSON import for reliable transfer.
