Overview

The Character Vault is a roster management tool for Savage Worlds characters. It's designed for game masters, writers, and publishers who need to maintain large collections of NPCs, pre-generated characters, and named personalities across multiple products and regions.

Where the Character Forge builds one character at a time from scratch, the Vault manages the fleet ‚Äî dozens or hundreds of characters across a product line, each with mechanical builds, narrative content, and production status tracking.

The Vault saves automatically to a GitHub Gist (cloud storage) with every edit. Your data persists across sessions, devices, and browser refreshes. No manual saving is required.

Quick Start

First visit: the Vault will prompt you to enter a GitHub Personal Access Token with Gist permissions. This connects the tool to your private cloud storage. You'll only need to do this once per browser.

Once connected, you can create characters with + New Character, or import existing builds via ‚áÑ Import. Each character appears in the sidebar list and can be selected for editing.

The three-column layout shows your roster on the left, the active editing panel in the middle, and the character's stat summary on the right. Click any section heading on the right to jump into its editor.

Cloud Storage (Gist)

The Vault stores all character data in a GitHub Gist ‚Äî a private, version-controlled JSON file hosted on GitHub's infrastructure. This gives you cloud persistence, device portability, and a complete edit history accessible through GitHub's web interface.

Setup

On first use, the Vault asks for a GitHub Personal Access Token. Generate one at github.com/settings/tokens with the gist scope. The token is stored in your browser's localStorage and never sent anywhere except GitHub's API.

How Saving Works

Every edit triggers an automatic save cycle: a 500ms debounce collects rapid changes, then the full roster is serialized and pushed to the Gist. The header sync indicator shows the result ‚Äî green for success, red for failure.

Saves happen silently in the background. There is no save button and no save overlay during normal operation. If a save fails (network issues, token expiry), the indicator turns red and the error is logged to the browser console.

Version History

Because the Gist is a Git repository, every save creates a new revision. You can view the full edit history at https://gist.github.com/<gist-id>/revisions. GitHub stores these revisions indefinitely, providing a free backup system.

If you ever need to roll back to a previous version, you can view the Gist revision history on GitHub and copy an earlier version of the JSON data.

Character Roster

The left column displays your complete character roster. Each entry shows the character's name, ancestry, region, and tier. Status indicators appear as coloured dots or checkmarks for completion milestones.

Click a character to select it and load their data into the right column and editing panels.

Roster Stats

The header bar shows aggregate production stats: total characters, how many have their stat block complete, how many have narrative content finished, and how many are marked as Fantasy Grounds-ready. These update in real time as you edit.

Creating Characters

Click + New Character in the sidebar to open the creation modal. Required fields:

NameCharacter's name. Must be unique within the roster.
AncestryDropdown of available ancestries (Human, Dwarf, Elf, etc.).
RegionWhere this character is from ‚Äî Ammaria, Saltlands, Vinlands, Concordium, or Glasrya.
TierWild Card or Extra. Affects bennies and the Wild Die.
CategoryNPC, Pre-gen, Named NPC, Villain, Creature, etc. For filtering and organisation.

Optional fields: concept, product reference code. These can be filled in later via the Edit panel.

New characters start with all attributes at d4, no skills, and default derived stats. Build them up using the editing panels.

Editing Characters

Select a character, then click any section heading on the right column to open its editor. Alternatively, click the Edit button in the action bar for the core stats panel, or use the quick-link buttons in the middle column.

All edits save automatically. There is no save button ‚Äî the moment you change a value, it's written to memory and queued for cloud sync. You can edit freely without worrying about losing work.

The ‚úé icon next to section headings indicates that section is clickable and will open an editor panel.

Search & Filtering

The search bar at the top of the roster filters characters by name. Type to filter ‚Äî matches are instant. Clear the search to show all characters.

Filter dropdowns below the search bar let you narrow by Region, Category, and Tier. Filters are cumulative ‚Äî selecting both "Ammaria" and "Wild Card" shows only Wild Card characters from Ammaria.

The Status Dashboard button switches to a table view showing all characters with their completion status in a sortable grid.

Panel ‚Äî Skills

Lists all assigned skills with die stepper buttons. Click + to add a new skill from the dropdown, or type a custom skill name. Skills are sorted alphabetically.

Each skill shows its current die value. Use the ‚àí and + buttons to step up or down. The die chain is d4‚Üíd6‚Üíd8‚Üíd10‚Üíd12. Pre-gen characters are capped at d12.

Click the ‚úï button next to a skill to remove it entirely.

Core skills (Athletics, Common Knowledge, Notice, Persuasion, Stealth) typically start at d4 for player characters. For NPCs, only assign the skills that character would actually use ‚Äî there's no point budget for NPCs.

Panel ‚Äî Hindrances

Select hindrances from the catalogue dropdown or type a custom hindrance name. Each hindrance is added to the character's list and displayed in the stat summary.

Include the severity in the name for clarity ‚Äî e.g. "Loyal (Minor)" or "Vengeful (Major)". The Build Audit checks hindrance count against Savage Worlds limits.

Panel ‚Äî Edges

Select edges from the catalogue or type a custom edge name. The catalogue contains all core Savage Worlds edges plus any edges from enabled setting modules.

Edges not found in the catalogue are tagged as (legacy) in the stat summary ‚Äî this flags them for review but doesn't prevent their use. This is common when importing characters built under older rulesets.

Panel ‚Äî Weapons

Add weapons from the equipment catalogue. Each weapon entry includes name, damage, AP value, and range (if applicable). The catalogue covers all Savage Worlds core weapons.

Weapons appear in the stat summary with their full stats and are included in all exports. Weapons with apostrophes in their names (Sailor's Cutlass, etc.) are handled correctly.

Panel ‚Äî Armour

Add armour from the catalogue. Each piece shows name, protection value, and location coverage. When armour is added or removed, the character's Toughness armour bonus is recalculated automatically.

The Toughness display updates in real time ‚Äî "7 (2)" means base Toughness 7 with 2 points of armour. The stat summary on the right column reflects this immediately.

Panel ‚Äî Gear

General equipment that isn't a weapon or armour ‚Äî rope, lanterns, backpacks, trade goods, etc. Enter items as a comma-separated text list. The gear text is displayed as-is in stat blocks and exports.

This is a legacy field from earlier versions of the Vault. Characters imported from older data may have their weapons and armour listed here as text rather than in the structured Weapons and Armour panels. If so, add them properly via those panels and clear the gear text.

Stat Block

The Stat Block panel renders a formatted, read-only view of the character in the standard Savage Worlds stat block format. This is the "what it looks like in the book" view ‚Äî attributes on one line, skills on the next, derived stats, then edges, hindrances, gear, and powers.

Use this to proofread before export or Canon Commit. The layout matches Pinnacle Entertainment's published format for pre-generated characters.

Status Dashboard

Click the Status button in the header to switch to a table view showing every character's production status. Columns include: ref code, name, category, region, tier, build legal status, canon committed, stat complete, narrative complete, and FG-ready.

Use this view to identify which characters still need work. The three completion checkboxes (Stats, Narrative, FG Ready) are manually toggled on each character's action bar ‚Äî check them when you're satisfied that section is finished.

Canon Commits

The Canon Commit system provides a formal "sign-off" mechanism for finished characters. When you click üì¶ Commit to Canon, the Vault takes a snapshot of the character's current state and records it as the canonical version.

The commit modal shows a diff of the character's data ‚Äî attributes, skills, edges, hindrances, weapons, armour, derived stats, and equipment funds. Review the diff, then click üîí Commit to Canon to confirm.

Committed characters show a üì¶ badge in the Status Dashboard. The commit timestamp is recorded for audit trail purposes.

Canon Commits are a development workflow tool. The commit data is stored in the Gist alongside the character roster. A future Canon API endpoint will accept these commits for external validation, but this is not yet implemented.

Export Formats

Open the ‚áÑ Transfer panel on any character to access export. Four formats are available, each with Copy to Clipboard and Download File buttons:

JSON (Recommended)

Lossless. Complete character data in diceforge-character-v1 format. Use for Forge transfer, archiving, collaboration, or backup. The only format that round-trips perfectly.

Plain Text

Formatted stat block in the standard Savage Worlds layout. For pasting into documents, emails, or chat. Not reliably re-importable.

HTML

Self-contained styled web page with Georgia serif type, dark red headings, proper stat block layout. Opens in any browser. Print-friendly. Import is experimental.

RTF

Rich Text Format ‚Äî opens in Word, LibreOffice, Google Docs. Bold labels, coloured headings, formatted sections. Import is experimental.

Import

Click ‚áÑ Import in the sidebar to open the import modal. Two input modes:

Paste Text

Paste character data directly into the text area. The Vault auto-detects the format: JSON objects starting with { get lossless parsing; HTML or RTF markup is stripped and fed through the stat block parser; plain text goes through the stat block parser directly.

Upload File

Click to browse or drag a file. Accepts .json, .txt, .html, and .rtf. The file's content is loaded into the text area and auto-detected as above.

Preview & Validate

Click üëÅ Preview to parse the data without committing. The preview shows: name, ancestry, tier, category, attributes, skills, edges, hindrances, weapons, and armour. If the format is non-JSON, a yellow warning appears reminding you to review all fields.

If a character with the same name already exists, a duplicate warning appears. You can still import ‚Äî it creates a second character with the same name.

Non-JSON imports are best-effort. The stat block parser looks for lines starting with "Attributes:", "Skills:", "Edges:", "Hindrances:", "Pace:", "Gear:", etc. Weapons and armour listed as text will not be parsed into structured data ‚Äî add them via the Weapons and Armour panels after import.

JSON Transfer

The Vault and the Forge share the diceforge-character-v1 JSON schema. This enables lossless bidirectional transfer between the two tools.

Vault ‚Üí Forge

Open a character's Transfer panel ‚Üí select JSON ‚Üí Copy or Download ‚Üí Go to the Forge ‚Üí Concept step ‚Üí Upload JSON or Paste JSON.

Forge ‚Üí Vault

In the Forge, go to the Export step ‚Üí click JSON (Vault Transfer) ‚Üí Download or copy ‚Üí Go to the Vault ‚Üí ‚áÑ Import ‚Üí paste or upload the file.

Field Mapping

The core build transfers perfectly: name, concept, ancestry, attributes, skills, edges, hindrances, powers. Some fields require awareness:

Skills format: The Forge uses an array of {name, die} objects; the Vault uses a {name: die} object. Both tools convert automatically on import.

Gear: The Forge stores all equipment in a single unified array with type flags (dmg for weapons, armor value for armour). The Vault separates weapons, armour, and general gear into three fields. Conversion maps damage-bearing items to weapons, armour-value items to armour, and everything else to gearText. Costs may need re-entry as the tools track them differently.

Narrative fields: The Forge doesn't display quote, background, motivation, secret, services, adventureHook, or tactics ‚Äî but these fields survive a Forge round-trip in the JSON data, so Vault‚ÜíForge‚ÜíVault won't lose them.

Equipment Catalogues

The Vault ships with equipment catalogues for Weapons, Armour, and Gear drawn from the Savage Worlds core rules. Catalogue items include correct damage values, armour ratings, costs, weights, and AP values.

You can also type custom item names that aren't in the catalogue. Custom items are stored as-is ‚Äî you'll need to enter damage, armour, and other values manually.

Items with apostrophes (Sailor's Coat, Knight's Shield, etc.) are handled correctly throughout the tool ‚Äî an HTML entity escaping function prevents display issues in catalogue dropdowns and stat summaries.

Troubleshooting

Cloud sync indicator is red

This means a save to GitHub Gist failed. Common causes: expired or revoked Personal Access Token, network connectivity issues, or GitHub API rate limiting. Check the browser console (F12 ‚Üí Console) for the specific error. If your token has expired, generate a new one and clear/re-enter it in the Vault's setup prompt.

Characters don't appear after refresh

The Vault loads from the Gist on page load. If the Gist is empty or the token is wrong, the roster will be empty. Check that your token has gist scope and that the Gist ID is correct. You can view your Gist directly at https://gist.github.com/<your-gist-id>.

Weapons or armour disappear on reload

This was a known bug in early versions where the Gist serialization omitted weapon and armour arrays. If you encounter this on an older version, hard-refresh (Ctrl+Shift+R) to load the latest code. Weapons and armour now persist correctly.

Build Audit shows red but the character looks right

The audit is calibrated for Novice-rank characters. Higher-rank characters will naturally exceed Novice budgets. NPCs and creatures don't follow player creation rules. Read the specific audit message ‚Äî it tells you what's expected vs. actual. Warnings (yellow) are informational; only failures (red) indicate a genuine rules issue.

Apostrophe items won't add from catalogue

Fixed in the current version. Hard-refresh if you're still seeing this. The HTML entity escaping now handles single quotes correctly.

Import shows "No attributes or skills found"

The stat block parser expects lines formatted as "Attributes: Agility d6, Smarts d8..." and "Skills: Fighting d8, Shooting d6...". If your source text uses a different format (colons, em-dashes, tabs), the parser may not recognise it. Use JSON import for reliable transfer.
